# AI 开发 0 到 1 — 教程网站设计方案

## 项目概述

为零编程基础的用户提供一份基于 Cursor IDE 的 Vibe Coding 实践教程。通过循序渐进的项目实践，让学员从"和 AI 聊天做东西"开始，逐步掌握 MCP、Agent Skills 等进阶工具，最终具备独立使用 AI 开发应用的能力。

## 目标读者

完全零基础：不知道什么是 HTML/CSS/JS，没用过终端/命令行。

## 核心设计原则

1. **需求驱动工具**：不是"现在来学 MCP"，而是"我们想实现 XX 功能，有个工具可以让这件事变简单"
2. **自然开发节奏**：不预设固定课时，按实际 AI 开发的自然流程组织内容
3. **不同 prompt 不同效果**：在合适的地方展示不同提示词带来的不同结果，让学员建立"怎么说决定 AI 怎么做"的核心认知
4. **正常流程 + 常见问题**：每个功能模块展示顺畅的主流程，另附常见问题供遇到问题时查阅
5. **AI 即排错工具**：教会学员遇到问题时让 AI 帮忙排查，而不是试图穷举所有错误场景

## 技术方案

- **框架**：VitePress（文档框架，开箱即用的导航、搜索、暗色模式等）
- **部署**：静态部署（adapter-static），后续如需动态功能可迁移至服务端部署
- **内容格式**：Markdown 图文，截图占位符由开发者预设路径和 alt 描述，后期由作者截图覆盖

## 课程结构

### 总体规划

两个独立项目，技能递进但代码不耦合：

| 项目 | 技术栈 | 工具级别 | 核心学习目标 |
|------|--------|----------|-------------|
| 番茄时钟 | 纯 HTML/CSS/JS | 纯聊天式 Vibe Coding | 学会和 AI 对话做东西 |
| 贪吃蛇 | Vite + JS | 引入 MCP + Agent Skills | 掌握进阶工具提升开发效率 |

后续可根据学员反馈追加第三个商用级品质的项目（如个人记账应用），暂不纳入首期内容。

### 每个内容页的固定格式

1. **本节目标**：一句话说清楚这一节要做什么
2. **正常流程**：prompt 示例 → AI 的响应描述 → 结果截图
3. **提示词对比**（在合适的地方）：展示不同 prompt 的效果差异
4. **常见问题**：如果出现了 XX 情况，可以这样让 AI 帮你处理

## 项目一：番茄时钟

### 定位

学员的第一个项目，全程纯聊天式 Vibe Coding，不引入任何进阶工具。

番茄时钟使用纯 HTML/CSS/JS，无需安装 Node.js，浏览器直接打开 HTML 文件即可运行，真正的零门槛起步。

### 核心学习目标

- 学会用自然语言描述需求给 AI
- 学会查看和接受 AI 生成的代码
- 学会遇到问题时让 AI 帮忙排查
- 理解"不同提示词 → 不同结果"的核心概念
- 建立"我能做出东西"的信心

### 功能模块

1. **创建项目**：告诉 AI "帮我用纯 HTML、CSS 和 JavaScript 创建一个番茄时钟网页应用"，从零到第一次看到页面跑起来
2. **实现倒计时**：25 分钟倒计时核心功能，能看到时间在走
3. **控制按钮**：开始、暂停、重置
4. **工作/休息切换**：25 分钟工作 + 5 分钟休息的自动切换
5. **视觉优化**：让 AI 美化界面，配色、动画、响应式
6. **提示音/通知**：时间到了有声音或浏览器通知

### 提示词教学点

在"创建项目"章节，展示两种 prompt 策略的对比：

**策略一（精确 prompt）**：
> "帮我用纯 HTML、CSS 和 JavaScript 创建一个番茄时钟网页应用，不使用任何框架，尽量简单"

AI 大概率直接开始生成代码，流程顺畅。

**策略二（模糊 prompt + 委托 AI 决策）**：
> "帮我做一个番茄时钟"

AI 可能追问技术栈选择（React/Vue 等），学员可回复：
> "我是零基础，请帮我选择最简单的方案"

通过对比让学员理解：精确的 prompt 能减少沟通轮次，模糊的 prompt 也能通过追加说明达到目的。

## 项目二：贪吃蛇

### 定位

第二个项目，在已掌握基础 Vibe Coding 的前提下，引入 Node.js 环境、构建工具，以及 MCP 和 Agent Skills。

### 核心学习目标

- 熟悉"有构建工具"的项目流程（npm、dev server）
- 理解 MCP 是什么、为什么有用
- 学会安装和使用 Agent Skills
- 体会"有工具辅助"和"纯聊天"的差异

### 功能模块

1. **环境升级**：安装 Node.js，用 AI 创建一个 Vite + JS 项目
2. **基础游戏**：蛇的移动、食物生成、碰撞检测
3. **游戏完善**：计分、速度递增、游戏结束/重新开始
4. **引入 MCP**：连接外部服务（如存储排行榜数据），让学员体会 AI 能直接操作外部工具
5. **引入 Agent Skills**：安装代码规范类 skill，体会 skill 带来的质量提升
6. **视觉打磨**：界面美化、动画效果、移动端适配

### 与番茄时钟的对比教学点

| 维度 | 番茄时钟 | 贪吃蛇 |
|------|---------|--------|
| 文件结构 | 单个 HTML 文件 | 有项目结构（src/、package.json 等） |
| 运行方式 | 浏览器直接打开 | npm run dev |
| AI 工具 | 纯聊天 | 聊天 + MCP + Agent Skills |
| 复杂度 | UI 交互为主 | 游戏逻辑 + 外部服务 |

## 协作流程

### 截图处理

- 开发者在 Markdown 中预设图片占位符，包含路径和 alt 描述
- 每个项目目录下有 `images/` 文件夹
- 作者按教程操作一遍，对照 alt 文本截图，用对应文件名保存
- 示例：`![创建项目后的Cursor界面](./images/pomodoro-01-create-project.png)`

### 内容校准

由于 AI 的输出不完全确定，作者实际操作时如果发现 AI 的回应与教程描述不一致，反馈给开发者调整教程内容。
